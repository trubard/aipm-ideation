# 변경 이력 (CHANGELOG)

이 워크스페이스의 **도구(명령어·양식) 변경**을 기록한다. 사용자의 아이디어 산출물은 각자 관리.
업데이트·마이그레이션 방법은 [README의 "업데이트 & 마이그레이션"](README.md#업데이트--마이그레이션) 참고.

형식: 최신이 위. 날짜는 `YYYY-MM-DD`. 영향은 🟢 추가(호환) / 🟡 변경(주의) / 🔴 파괴적(작업 필요)로 표시.

## 2.3 — 2026-06-17

### 🔴 파괴적 (작업 필요)
- **`structure/`(시스템) ↔ `my-idea/`(내 작업) 분리** — 단계 폴더(`10_아이디어풀`·`20_탐색중`·`30_검증정의`·`99_아카이브`)와 `90_템플릿`을 **`structure/` 아래 빈 골격**으로 이동(읽기 전용 시스템). 실제 산출물은 **`structure/`에서 복제한 `my-idea/`**(`.gitignore` — 공유 제외)에서 관리. 시스템과 개인 데이터가 분리돼 `git pull` 충돌이 사라지고, 초기화는 `my-idea/`만 리셋하면 된다.
  - 5개 슬래시 명령어를 전부 갱신: 산출물은 `my-idea/...`에 쓰고, 양식은 `structure/90_템플릿/...`에서 읽는다.

### 🟢 추가
- **`/초기화` 명령어** — `structure/` 골격에서 `my-idea/`를 생성하거나, 내 산출물을 비우고 빈 골격으로 리셋한다(리셋은 확인 필수, `structure/`는 건드리지 않음). 셸 명령 대신 쓸 수 있다.
  - 마이그레이션: clone/pull 후 `my-idea/`가 없으면 README의 "최초 1회" 명령으로 생성. 이전 버전에서 루트에 만든 아이디어 폴더는 `my-idea/<단계>/` 아래로 이동.

## 2.1.1 — 2026-06-16

### 🟡 변경
- **샘플 아이디어 폴더 제거 + 백로그 초기화** — `20_탐색중/부모통화케어`·`세계관집필플랫폼`을 추적 대상에서 삭제하고, 전체 산출물은 **로컬 백업 폴더 `_백업/샘플_아이디어/`(.gitignore — 공유 제외)** 로 보존. [백로그](10_아이디어풀/아이디어_백로그.md)는 진행중 표·원석을 비운 **빈 출발 상태**로 초기화. 도구·양식과 사용자 작업 영역을 분리해 pull 충돌·샘플 혼입을 제거.
  - 마이그레이션: 예전 버전에서 샘플 폴더를 그대로 뒀다면 pull 시 삭제됨. 내 아이디어 폴더(다른 이름)는 영향 없음. 샘플 내용은 받는 쪽엔 포함되지 않으므로(원작자 로컬에만 백업) 필요하면 별도 공유.
